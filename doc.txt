Ceci est une structure de site afin de débuter un site plus rapidement. Pour l'utiliser, vous devez travailler en ligne ou sur un serveur local !!!

j'ai testé Bootstrap, foundation, mais s'ils sont parfait pour styler les pages d'administration, jamais j'ai pu m'en servir pour styler un site dont le design est trop différent de ce que propose les frameworks. Cette structure a un rendu très basique pour que vous puissiez ne penser qu'à personnaliser et non pas bidouiller pour que ça soit pareil sous Firefox, IE, Chrome, etc..

Le "htaccess" fourni est celui de bootstrap qui est vraiment hyper travaillé, j'y ai ajouté des lignes spécifiques à OVH en haut que vous pouvez effacer si vous êtes hébergé ailleurs. Il faut renommer le fichier en ".htaccess" une fois mis sur le FTP !!!
Ce fichier est facultatif, il est juste là pour optimiser le serveur et si vous rencontrez des difficultés, effacez le tout simplement.

jQuery 1.10.x a retiré les fonctions rendant les scripts compatibles avec les anciens navigateurs. Si des plugins refusent de fonctionner, utilisez jquery-migrate (voir site jQuery).

Prefixfree permet de ne pas avoir à ajouter les préfixes en CSS3 (-webkit-, -moz-…).

Modernizr permet de détecter si le navigateur ne supporte pas certaines choses en CSS ou HTML5 (se référer à la documentation sur le site). Par exemple, cela permet de mettre des effets de transitions en CSS3 et si le navigateur ne le supporte pas, de charger un fichier js qui le fera à la place.



CONSIGNES
---------
Les images de design vont dans "css/img" (background, logo, icônes, déco,…).

Les photos, documents entrant dans le contenu des pages vont dans "data" (y créer les sous-dossiers selon ses propres besoins).

Le meta "author" ne sert à rien, sauf à vous faire connaître d'autres développeurs/intégrateurs qui iront regarder les sources de la page ;-).

CSS
---
Le nom des classes est normalement parlant. Elles sont combinables. Par exemple :
<div class="left w10 aligncenter">lorem ipsum</div>
Le bloc flottera à droite, fera 10% de large et le contenu sera centré.

La partie RESET n'a pas besoin d'être modifiée.

La partie CLASSES UTILITAIRES peut être personnalisée selon votre design.

La partie STYLE CUSTOMS est là pour caser vos styles personnalisés !
En exemple, j'ai mis quelque chose de très utilisé par Facebook, Twitter et autre, les "medio box" : c'est pour styler des commentaires avec une image avatar à gauche.

La partie STYLES DEMO peut être effacée, c'est juste pour avoir une page présentable :-p.

La page HTML comprend des exemples sur l'utilisation des classes (regardez le code source).
